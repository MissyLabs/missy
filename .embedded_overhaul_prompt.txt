SECURITY REQUIREMENTS

- Default-deny mindset for privileged actions.
- Web TUI access, tool creation, tool execution, Discord, and scheduled
  execution must be safe by default.
- Secrets must come from environment or protected local files and be redacted.
- Network, filesystem, shell, plugin, provider, and Discord actions must go
  through policy where applicable.
- Dangerous actions need approval or explicit policy.
- Audit logs must record privileged actions and denials.
- Defend against prompt injection, malicious generated tools, tool supply-chain
  abuse, Web UI auth bypass, CSRF, XSS, data exfiltration, SSRF, scheduler
  abuse, secrets leakage, Discord spoofing, bot loops, impersonation, unsafe
  attachments, confused channel routing, benchmark gaming, and unsafe
  provider-specific tool enablement.
